# 01｜《星渊誓约》项目概述与 MVP 游戏设计文档

> 版本：MVP Baseline 1.0  
> 目标关卡：断裂点三号  
> 本文将原开发规划中未明确但实现所必需的规则补充为项目基线。

## 1. 游戏定位

《星渊誓约》是一款原创幻想题材、回合制网格战棋 RPG。

MVP 的设计重点不是内容量，而是验证以下差异化玩法：

- 实相与星相双层地块；
- 玩家消耗 PV 翻转地块相位；
- 相位变化引发坠落或挤压；
- 激活锚点形成几何围区；
- 围区改变物理防御规则；
- 律令在移动步骤中插入反应结算；
- 玩家可撤销 Command，但会提高 CV；
- 全部规则可确定性回放。

## 2. 目标平台与范围

MVP 首发验证平台：

```text
Windows PC
```

技术平台：

```text
Unity 6.3 LTS
Universal Render Pipeline
C#
```

MVP 不追求：

- 商业化；
- 长线养成；
- 多关卡内容；
- 高品质正式美术；
- 联机；
- 正式存档。

## 3. 核心体验目标

玩家应感受到：

1. 地形不是静态障碍，而是可主动翻转的战术资源；
2. 翻转地形可能救人，也可能造成坠落和挤压；
3. 锚点位置决定围区，而围区会改变伤害结算；
4. 律令能够打断敌人的移动计划；
5. Undo 可以修正操作，但会推动世界坍缩；
6. 规则透明、预览与结果一致，不依赖隐藏随机。

## 4. 核心术语

### Reality / 实相

地块的物质层。

### Astral / 星相

地块的星渊层。

### ActiveLayer

地块当前激活层。单位能否站立由单位层、地块激活层和该层可通行性共同决定。

### PV：Phase Value

团队相位资源。

- 初始值：3；
- 最大值：3；
- MVP 无被动恢复；
- 每次合法 `PhaseFlipCommand` 消耗 1；
- PV 不足时不可翻转。

### CV：Collapse Value

坍缩值，代表撤销现实造成的世界不稳定。

- 初始值：0；
- 每次成功 Undo 增加 10；
- 失败 Undo 不增加；
- 达到 `CollapseThreshold=100` 时立即失败；
- HUD 必须持续显示。

### Command

所有会影响战斗状态的玩家或敌方行为。

### BattleEvent

用于规则链和回放验证的核心事件。

### PresentationEvent

用于 Unity 动画、音效和 UI 的表现事件，不参与战斗真值。

## 5. 棋盘与坐标

MVP 地图：

```text
宽度：8
高度：10
格子数：80
```

坐标：

- 左下角为 `(0,0)`；
- `x` 向右；
- `y` 向上；
- 稳定排序先 `y`，后 `x`。

相邻格只包括四方向：

```text
下 (0,-1)
左 (-1,0)
右 (1,0)
上 (0,1)
```

对角线不算相邻。

## 6. 单位与层

每个单位至少具有：

```text
UnitId
TeamId
LayerId
Position
LastLegalPosition
MaxHP
HP
POW
ARC
ARM
RES
MOV
MaxAP
CurrentAP
Statuses
IsDefeated
```

MVP 中单位的 `LayerId` 固定，不提供主动切换单位层的技能。

一个格子对单位合法，需要同时满足：

1. 坐标在棋盘内；
2. 地块 `ActiveLayer == Unit.LayerId`；
3. 该层为可通行；
4. 没有其他存活单位占用；
5. 未被关卡规则禁止进入。

## 7. 回合与行动点

战斗按队伍回合运行：

```text
Player Turn
→ Enemy Turn
→ Round + 1
→ Player Turn
```

规则补全：

- Round 从 1 开始；
- 玩家队伍先行动；
- 每个存活单位在己方回合开始时恢复至 `MaxAP=2`；
- 单位 AP 归零后仍可被选择查看，但不能执行消耗 AP 的 Command；
- `EndTurnCommand` 结束整个队伍回合，不要求所有单位 AP 用尽。

Command 消耗：

| Command | AP | 其他资源 |
|---|---:|---|
| Move | 1 | 路径长度不超过 MOV |
| Attack | 1 | 无 |
| DeployDecree | 1 | 消耗对应律令实例 |
| PhaseFlip | 0 | 1 PV |
| Undo | 0 | 成功后 +10 CV |
| EndTurn | 0 | 结束队伍回合 |

## 8. 移动

`MoveCommand` 必须提供完整路径。

合法性：

- 单位存在且存活；
- 属于当前行动队伍；
- 起点等于当前位置；
- 路径每一步四方向相邻；
- 步数不超过 MOV；
- 单位至少有 1 AP；
- 每一步地块合法且未占用；
- 不得穿越非法格。

执行：

1. 扣除 1 AP；
2. 逐步移动；
3. 每一步更新 `LastLegalPosition` 为进入前的合法位置；
4. 每一步发布 `MoveStep`；
5. 每个 `MoveStep` 立即进入 ReactionResolver；
6. 若律令改变位置、施加阻止移动的状态或单位死亡，后续路径停止。

寻路预览和敌方 AI 使用相同的稳定邻居顺序。

## 9. 相位翻转

`PhaseFlipCommand` 目标为一个地块。

合法性：

- 当前为玩家回合；
- 坐标存在；
- PV > 0；
- 地块带 `PhaseMutable`；
- 目标地块与至少一个存活玩家单位的曼哈顿距离不超过 3；
- 战斗未结束。

执行：

1. 切换 `ActiveLayer`；
2. PV -1；
3. 按 `UnitId` 检查该格上的单位；
4. 对非法占用执行挤压或坠落；
5. 发布规则事件；
6. 写入历史。

## 10. 挤压与坠落

### 挤压

翻转后单位当前格非法，且 `LastLegalPosition`：

- 在棋盘内；
- 对该单位合法；
- 未被占用；

则：

1. 单位回退；
2. 施加 `Stunned(1)`；
3. 发布位移和状态事件。

### 坠落

无法挤压回退时：

1. BFS 搜索最近合法格；
2. 距离优先；
3. 同距离按 `y`、`x`；
4. 移动到目标格；
5. 受到 `floor(MaxHP × 20 / 100)` 伤害；
6. 若 HP ≤ 0，则单位被击败并移出占用查询。

### 无合法格补全规则

若棋盘上不存在合法格：

- 单位立即被判定为 `Defeated`；
- 从棋盘占用中移除；
- 发布 `UnitLostToVoid`；
- 不再创建额外复杂死亡机制。

多单位同时处理时按 `UnitId`。

## 11. 攻击与伤害

技能分为：

```text
Physical
Magical
```

基础伤害：

```text
baseDamage = floor(skillPower × attackStat / 100)
```

物理：

```text
attackStat = POW
defenseValue = ARM
```

法术：

```text
attackStat = ARC
defenseValue = RES
```

坍缩围区内的物理攻击：

```text
defenseValue = 0
```

防御因子：

```text
defenseFactor = floor(10000 / (100 + defenseValue))
```

最终伤害：

```text
finalDamage = max(1, floor(baseDamage × defenseFactor / 100))
```

修正顺序：

1. 读取技能和攻击属性；
2. 读取防御；
3. 应用围区规则；
4. 计算基础最终伤害；
5. 应用 Guarding；
6. 应用 Marked；
7. 最低伤害 1；
8. 扣减 HP；
9. 处理击败。

预览与实际结算必须调用同一个 `DamageResolver`。

## 12. 状态效果

### Stunned(n)

- 在单位接下来 `n` 个己方回合无法执行任何消耗 AP 的 Command；
- 回合开始 AP 设为 0；
- 对应回合结束后剩余回合 -1。

### Paralyzed(n)

- 在接下来 `n` 个己方回合不能执行 Move；
- 可以攻击或部署律令；
- 对应回合结束后剩余回合 -1。

### MoveZeroNextTurn

- 下一个己方回合 MOV 视为 0；
- 该回合结束后移除。

### Guarding

- 受到攻击的最终伤害降低 25%，向下取整；
- 最低伤害仍为 1；
- 在单位下一个己方回合开始时移除。

### Marked

- 下一次受到成功攻击时，最终伤害提高 20%，向下取整；
- 本次伤害结算后移除。

相同状态默认刷新持续时间，不叠加数值；不同实例按稳定规则合并。

## 13. 锚点与围区

锚点地块带 `Anchor` 标签。

激活规则补全：

- 玩家单位完成移动并停在未激活锚点上时自动激活；
- 不额外消耗 AP；
- 敌方单位不能激活玩家锚点；
- 激活后本场战斗永久保持，Undo Replay 除外。

有效围区：

- 使用全部已激活锚点；
- 枚举三点和四点组合；
- 排除重复、共线和自交；
- 边界算内部；
- 多个有效围区取面积最大；
- 面积相同按规范化顶点序列字典序最小者；
- 围区内地块和单位结果必须稳定。

围区效果：

- 只对物理攻击生效；
- 围区内的被攻击单位 ARM 视为 0；
- 不影响 RES；
- 不额外造成持续伤害。

## 14. 律令

MVP 只有：

```text
引力律令·崩裂
```

部署规则补全：

- 玩家队伍拥有 1 个实例；
- 由存活玩家单位执行；
- 消耗 1 AP；
- 部署距离为曼哈顿 3；
- 目标地块必须带 `DecreeDeployable`；
- 目标格不能已有律令；
- 部署后直到触发或战斗结束。

触发：

- 敌方 `MoveStep` 进入律令格；
- 同一 MoveStep 最多触发一个律令；
- 先结算律令，再决定原移动是否继续；
- 成功触发后律令失效。

效果：

```text
PerpendicularKnockback
distance = 2
fallbackStatus = Paralyzed(1)
```

垂直方向补全：

1. 根据敌人进入该格的移动向量生成左、右两个垂直方向；
2. 分别模拟最多 2 格合法击退距离；
3. 选择可移动格数更多的方向；
4. 若相同，选择最终坐标按 `y`、`x` 更小的方向；
5. 若两个方向都不能移动至少 1 格，施加 `Paralyzed(1)`；
6. 击退中不触发同一律令第二次，但产生的 MoveStep 可供其他允许的反应系统使用。

## 15. Undo 与 CV

Undo 通过删除历史中的最后一条可撤销玩家 Command，并从初始状态 Replay 其余 Command 实现。

可撤销：

- Move；
- Attack；
- PhaseFlip；
- DeployDecree。

不可撤销：

- EndTurn；
- 敌方 Command；
- Undo；
- 已结束战斗中的 Command；
- 上一个玩家回合之前的 Command。

成功 Undo：

- 删除一条 Command；
- Replay；
- CV +10；
- 生成 Undo PresentationEvent；
- Undo 自身不进入普通历史。

失败 Undo：

- 不改状态；
- 不增加 CV；
- 返回明确错误。

## 18. 敌方 AI

敌方按 `UnitId` 顺序行动。

每个敌人：

1. 若存在可攻击目标，选择；
2. 否则向目标移动；
3. 移动后若可攻击，再攻击；
4. AP 用完后处理下一个单位。

目标选择优先级：

1. 可在本回合击败；
2. 当前 HP 更低；
3. 曼哈顿距离更近；
4. `UnitId` 更小。

移动目标：

- 防守阶段：优先接近防守区域内玩家，其次最近玩家；
- 撤离阶段：优先阻挡撤离区域，其次最近玩家。

路径使用统一确定性寻路，不使用随机。

## 19. 断裂点三号

地图坐标见 `03_Data_and_Content_Spec.md`。

### 阶段一：防守

防守区域为 2×2 区域。

每个玩家回合结束时：

- 至少一个存活玩家单位在防守区域内：同步进度 +1；
- 无单位在区域：进度不增加；
- 进度达到 3：切换撤离目标。

同步进度不会倒退。

### 阶段二：撤离

胜利条件：

- 所有存活玩家单位进入撤离区域；
- 且当前没有待结算 Reaction；
- 立即触发 `BattleWon`。

失败条件：

- 所有玩家单位被击败；
- CV ≥ 100；
- Round 超过 12；
- 配置或状态进入不可恢复的规则错误。

## 20. MVP 完整体验要求

完成关卡的推荐路径必须能覆盖：

- 移动；
- 攻击；
- 相位翻转；
- 坠落；
- 挤压；
- 激活至少 3 个锚点；
- 形成有效围区；
- 触发引力律令；
- 成功 Undo；
- 防守目标切换撤离；
- 最终胜利。
